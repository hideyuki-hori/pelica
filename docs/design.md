# pelica - UI 設計仕様

家計簿と支出シミュレーションを統合した個人向けデスクトップアプリ「pelica」の UI 設計仕様。

このドキュメントは Pencil でのデザイン作業および Solid 実装の参照資料。

---

## プロジェクト概要

- **形態**: Tauri デスクトップアプリ（WebView ベース、 Solid + TypeScript）
- **主用途**: PC 操作（スマホ対応は将来検討）
- **言語**: 日本語 UI
- **通貨**: 円（JPY）固定
- **ウィンドウサイズ**: 1200×800 想定（responsive 不要）

---

## 設計思想（最も重要）

### 時系列ストリーム + フィルタ

- 取引データは「区切りのない連続したストリーム」として保持
- 月・年などの区切りはデータ側に持たない、 フィルタの段階で初めて発生
- 過去（実績）と未来（予定・予測）はストリーム上の連続体として扱う

### What-if スイッチ

- 各予定項目・予測項目は個別に on/off 可能
- 「もしこの支払いがなければ」「この収入が増えたら」を即座に確認できる
- 家計簿機能とシミュレーション機能を一体化させる中核仕様

---

## 機能仕様

### 取引種別
- 収入 / 支出 / 振替 の 3 種類
- 振替は from 口座 / to 口座 / 金額の 1 取引で表現

### 口座
- 現金・銀行・カードを統一表現（種別: cash / bank / card）

### カード
- 締め日 / 支払日 / 引落口座
- リボ・分割払いをサポート
- 休日繰越（休日は翌営業日）

### ローン
- 住宅 / 自動車 / 教育 / 消費者金融などの月次返済型
- 元利均等のみ（MVP）、 固定金利運用（変動は将来対応）
- 一部繰上返済をサポート
- 残元本・残利息・残回数を管理

### range フィルタ構文
- `range since:2026-01-01 until:2026-01-31`
- `range last:30d` / `range month:2026-01` / `range year:2026` / `range week:2026-W05`
- 詳細は [`range-filter.md`](./range-filter.md) を参照

## 画面一覧

### 1. ダッシュボード（メイン画面）

最初に開く画面。「今・直近」 を一望できる。

**主要要素:**
- 全口座の残高サマリ（種別ごとに集計可能）
- range フィルタ（デフォルトは「今月」）
- 期間内の収支サマリ（収入・支出・振替）
- 期末残高予測（実績 + 支払予定 + 予測支出）
- What-if スイッチパネル（各予定・予測項目の on/off）
- 直近の取引リスト（5-10 件）
- カード支払予定の直近表示
- ローン残高の概要

### 2. 取引入力フォーム

頻繁に使う、 素早く入力できることが最重要。

**主要要素:**
- タブ: 収入 / 支出 / 振替（キーボードショートカット切替）
- 共通フィールド: 日付、 金額、 メモ
- 収入: 入金先口座、 カテゴリ
- 支出: 出金元口座、 カテゴリ
- 振替: from 口座、 to 口座
- カード払いの場合: 分割回数（1=一括 / 2 以上=分割）
- 保存: Enter / Cmd+S
- 連続入力モード: 保存後にフォームをクリアして次の入力へ

### 3. 取引一覧

range で期間指定して、 該当する取引を一覧表示。

**主要要素:**
- range フィルタ（共通コンポーネント）
- 口座フィルタ（複数選択可）
- カテゴリフィルタ（複数選択可）
- 種別フィルタ（収入 / 支出 / 振替）
- 一覧テーブル: 日付 / 種別 / 金額 / 口座 / カテゴリ / メモ / アクション
- 各行のアクション: 編集 / 訂正 / 取消（イベントとして記録）
- 集計フッター: 期間内合計（収入 / 支出 / 純額）

### 4. 口座管理

口座の CRUD。

**主要要素:**
- 一覧: 種別 / 名前 / 残高 / 通貨 / 状態（アクティブ / 閉鎖）
- 追加・編集モーダル: 種別、 名前、 通貨（MVP は JPY 固定で表示なし）、 初期残高
- 削除: 取引が紐づいている口座は閉鎖扱い（イベント `AccountClosed` を発火、 物理削除はしない）

### 5. カード管理

カードの CRUD + 直近の支払予定表示。

**主要要素:**
- 一覧: 名前 / 締め日 / 支払日 / 引落口座 / 直近の支払予定額
- 追加・編集モーダル: 紐づく口座、 締め日（1-31 or 末日）、 支払日（1-31 or 末日）、 引落口座
- 直近 3 ヶ月分の支払予定を card_payment_schedules から表示

### 6. ローン管理

ローン契約の CRUD + 繰上返済入力。

**主要要素:**
- 一覧: 名前 / 種別 / 元金 / 残元本 / 残回数 / 次回支払日 / 金利
- 追加: 元金、 金利、 期間（月数）、 開始日、 引落日、 引落口座、 種別
- 詳細画面: 返済予定の全期間を一覧表示（元金 / 利息 / 残元本の推移）
- 繰上返済入力: 金額、 日付、 期間短縮 or 金額軽減 を選択

### 7. カテゴリ管理

カテゴリの CRUD。

**主要要素:**
- 一覧: 名前 / 種別タグ（収入向け / 支出向け / 両用）
- 追加・編集・削除（使用中カテゴリの削除は警告 + ソフト削除）

## UI/UX 方針

### 一般

- シンプル、 情報密度高め（家計を素早く把握できることを優先）
- ダーク / ライトモード対応
- グラフは最小限（残高推移、 月別収支程度）

### キーボード操作

- PC 用途のためキーボードショートカット重視
- 主要画面遷移: Cmd+1 (ダッシュボード), Cmd+2 (取引入力), Cmd+3 (取引一覧), ...
- 取引入力: Tab で項目移動、 Enter で保存、 Esc でキャンセル
- 連続入力モード（保存後にフォームをクリア）は ON / OFF 切替可能

### 入力効率

- 取引入力は少数のキーストロークで完結すること
- カテゴリ・口座は autocomplete（よく使うものを上位表示）
- 金額入力: 数値キーパッドで完結

### 表示

- 金額は通貨記号 + 桁区切り（例: ¥1,234,567）
- 日付は YYYY-MM-DD（ISO 8601）
- 過去 / 確定の取引と、 予定 / 予測の取引は視覚的に区別（透明度・破線・色など）

### What-if 表現

- 予定 / 予測の項目には個別の on/off スイッチ
- スイッチ操作後、 残高予測がリアルタイムに更新される
- 「シミュレーションモード」 として一時的に切り替え、 リセット可能

## 技術制約

- **フレームワーク**: Solid + TypeScript
- **スタイリング**: 未確定（Pencil 設計時に決定）
- **コンポーネント**: 自前で書く方針、 依存ライブラリは最小限
- **アクセシビリティ**: ARIA 属性とキーボードナビゲーションは確保
